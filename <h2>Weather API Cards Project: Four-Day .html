Weather API Cards Project: Four-Day Plan

This project will be completed over four class sessions. On Day 1, students will focus on understanding JSON and getting the weather API to work in the browser without styling. On Day 2, students will style the weather cards with CSS. On Day 3, students will display an image based on weather conditions. On Day 4, students will display a wind icon based on wind speed using greater than and less than statements.

Project Files

Students should create a folder for the project with these files:

  • weather.html
  • weather-script.js
  • weather-styles.css
  • images folder for weather and wind icons

Day 1: Introduction to JSON and Displaying API Data

Day 1 Focus

The goal for Day 1 is to introduce students to JSON as a front-end data format and help them successfully fetch weather data and display it on the page without worrying about styling yet.

Day 1 Learning Objectives

  • Understand what JSON is and why it is used in front-end web development
  • Use fetch() to request data from a public API
  • Read data from arrays inside a JSON response
  • Use JavaScript to create and display weather cards
  • Display the date, high temperature, and low temperature for each day

Day 1 In-Class Work

  1. Introduce JSON and explain that APIs often return data as objects and arrays
  2. Show students the weather API URL and explain what it returns
  3. Open the API response and inspect the JSON structure
  4. Identify the specific weather arrays students will use: date, high temperature, and low temperature
  5. Build the JavaScript that fetches the data and loops through it
  6. Create a basic card for each day using JavaScript
  7. During class, add one additional weather node together as a group after students have date, high, and low working

Day 1 Required Output

By the end of Day 1, each student should have a working page that displays weather data in a simple unstyled or minimally styled format. Each card should show:

  • Date
  • High temperature
  • Low temperature
  • One additional weather item added during class

Day 1 Starter JavaScript

weather-script.js

const weatherContainer = document.getElementById(“weather-container”);

const apiUrl = “https://api.open-meteo.com/v1/forecast?latitude=52.52&longitude=13.41&daily=temperature_2m_max,temperature_2m_min,wind_speed_10m_max,wind_gusts_10m_max,rain_sum,precipitation_sum,sunrise,sunset¤t=temperature_2m,precipitation&timezone=auto&temperature_unit=fahrenheit”;

fetch(apiUrl)
.then(response => response.json())
.then(data => {
const dates = data.daily.time;
const maxTemps = data.daily.temperature_2m_max;
const minTemps = data.daily.temperature_2m_min;

for (let i = 0; i < 7; i++) {
const card = document.createElement(“div”);

const date = document.createElement(“h2”);
date.textContent = dates[i];

const maxTemp = document.createElement(“p”);
maxTemp.textContent = “High: “ + maxTemps[i] + “°F”;

const minTemp = document.createElement(“p”);
minTemp.textContent = “Low: “ + minTemps[i] + “°F”;

card.appendChild(date);
card.appendChild(maxTemp);
card.appendChild(minTemp);

weatherContainer.appendChild(card);
}
})
.catch(error => {
weatherContainer.innerHTML = “<p>Sorry, weather data could not be loaded.</p>”;
console.error(“Error fetching weather data:”, error);
});

Day 1 Teaching Notes

Keep the emphasis on reading the JSON and getting the page to work. Students do not need to worry about making it look good yet. The main goal is to understand the connection between the API response and the JavaScript output on the page.

Day 2: Styling the Weather Cards with CSS

Day 2 Focus

The goal for Day 2 is to take the working weather output from Day 1 and turn it into a visually organized layout using CSS.

Day 2 Learning Objectives

  • Use CSS to style repeated content consistently
  • Apply class names to dynamically created content
  • Use Grid or Flexbox to arrange weather cards on the page
  • Improve spacing, readability, and visual organization

Day 2 In-Class Work

  1. Add a class to each dynamically created card in JavaScript
  2. Create a layout for the cards using Grid or Flexbox
  3. Style the cards with background color, padding, spacing, and typography
  4. Add a hover effect or other visual improvement
  5. Discuss the separation of concerns: JavaScript creates the content and CSS controls the presentation

Day 2 JavaScript Update

Students should update the card creation code so each card gets a reusable class for styling.

Add this line after creating the card:

card.classList.add(“card”);

Day 2 Teaching Notes

Day 2 should focus on appearance and layout, not on adding lots of new weather data. Students should take the working content from Day 1 and improve how it looks. This reinforces the idea that content and presentation are two different parts of front-end development.

Homework After Day 2

  1. Add additional weather nodes to each card, such as wind speed, precipitation, rain total, sunrise, or sunset
  2. Convert the date into a more readable format
  3. Improve the visual design of the weather cards if needed

Day 3: Displaying an Image Based on Weather Conditions

Day 3 Focus

The goal for Day 3 is to help students understand how data can control visual output. Instead of only showing text such as date, high temperature, and low temperature, students will also display an image that matches the weather conditions.

Day 3 Learning Objectives

  • Read a weather condition value from the API response
  • Use conditional logic in JavaScript to decide which image to display
  • Create and insert image elements dynamically
  • Connect data values to visual design choices
  • Continue reinforcing the idea that JavaScript controls content and CSS controls presentation

Suggested Day 3 In-Class Flow

  1. Review the JSON structure and locate the weather condition field
  2. Explain what the weather condition values mean
  3. Create a simple system for mapping conditions to image filenames
  4. Use JavaScript to create an <img> element for each card
  5. Set the image source based on the weather condition
  6. Append the image to the weather card
  7. Style the images with CSS so they fit neatly into the card layout

Day 3 JavaScript Update

Make sure the daily weather code is included in the API request.

const apiUrl = “https://api.open-meteo.com/v1/forecast?latitude=52.52&longitude=13.41&daily=weather_code,temperature_2m_max,temperature_2m_min&timezone=auto&temperature_unit=fahrenheit”;

Then read the weather code array:

const weatherCodes = data.daily.weather_code;

Add image selection logic:

let imageFile = “images/cloudy.png”;

if (weatherCodes[i] === 0) {
imageFile = “images/sunny.png”;
} else if (weatherCodes[i] === 1 || weatherCodes[i] === 2 || weatherCodes[i] === 3) {
imageFile = “images/cloudy.png”;
} else if (weatherCodes[i] === 61 || weatherCodes[i] === 63 || weatherCodes[i] === 65) {
imageFile = “images/rainy.png”;
} else if (weatherCodes[i] === 71 || weatherCodes[i] === 73 || weatherCodes[i] === 75) {
imageFile = “images/snowy.png”;
}

Create and append the image:

const weatherImage = document.createElement(“img”);
weatherImage.src = imageFile;
weatherImage.alt = “Weather icon”;
weatherImage.classList.add(“weather-image”);

card.appendChild(weatherImage);

Day 3 CSS Update

Add this to weather-styles.css:

.weather-image {
width: 80px;
height: 80px;
object-fit: contain;
margin: 10px auto;
display: block;
}

Day 3 Teaching Notes

This is a good place to explain that APIs often return condition codes instead of plain English descriptions. Students may need to translate a weather code into a meaningful category such as sunny, cloudy, or rainy before choosing an image.

Day 4: Displaying Wind Icons Using Conditional Logic

Day 4 Focus

The goal for Day 4 is to help students understand how numeric data can drive different visual outcomes. Instead of matching exact values, students will evaluate ranges of values and display different icons based on those ranges.

Day 4 Learning Objectives

  • Use greater than (>) and less than (<) operators in JavaScript
  • Apply conditional logic to numeric data
  • Create different outputs based on value ranges
  • Reinforce how data controls visual presentation
  • Combine multiple dynamic elements within a single card

Example Wind Categories

  • Calm: less than 5 mph
  • Light wind: 5 to 10 mph
  • Moderate wind: 10 to 20 mph
  • Strong wind: greater than 20 mph

Day 4 In-Class Flow

  1. Review how wind data is retrieved from the API
  2. Display the wind speed value in each card
  3. Introduce greater than and less than operators
  4. Build a conditional structure to categorize wind speeds
  5. Assign an image file based on the category
  6. Create and append the wind icon to the card
  7. Style the icon with CSS

Day 4 JavaScript Update

Read wind speed data:

const maxWinds = data.daily.wind_speed_10m_max;

Create wind logic using comparisons:

let windImageFile = “images/wind-light.png”;

if (maxWinds[i] < 5) {
windImageFile = “images/wind-calm.png”;
} else if (maxWinds[i] < 10) {
windImageFile = “images/wind-light.png”;
} else if (maxWinds[i] < 20) {
windImageFile = “images/wind-moderate.png”;
} else {
windImageFile = “images/wind-strong.png”;
}

Create and append the wind icon:

const windIcon = document.createElement(“img”);
windIcon.src = windImageFile;
windIcon.alt = “Wind strength icon”;
windIcon.classList.add(“wind-icon”);

card.appendChild(windIcon);

Day 4 CSS Update

Add this to weather-styles.css:

.wind-icon {
width: 60px;
height: 60px;
display: block;
margin: 10px auto;
}

Day 4 Teaching Notes

This lesson introduces a more advanced concept than Day 3 because it uses ranges rather than exact matches. It is a good opportunity to stress that the order of conditions matters in if and else if statements.

Four-Day Project Summary

  • Day 1: Learn JSON, inspect the API response, and get the weather data working in weather-script.js
  • Day 2: Style the weather cards with weather-styles.css
  • Day 3: Display weather condition images using JavaScript and an images folder
  • Day 4: Use comparison logic to display wind strength icons

This four-day structure works well because students move from understanding data, to displaying data, to styling data, and then to using data to control visual design decisions in increasingly sophisticated ways.
